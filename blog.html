Scope of var, let, and const

The scope of a variable defines the region of your code where it’s accessible.

var: It is function scoped, meaning it’s visible throughout the function it is defined in, or global scope if defined outside any function.
let: It is block scoped, meaning it’s visible only within the closest {} block (function, if-else block, loop, etc.).
const: Similar to 'let', 'const' is block scoped. The difference is that the value assigned to 'const' can't be changed after it is assigned.

Use Cases of null and undefined

null: It is an assignment value. It can be assigned to a variable as a representation of no value or no object.
undefined: It is a type of variable that is not assigned any value. Variables that are declared but not initialized are of type 'undefined'.

Use Cases:
- Use 'null' when you want to intentionally represent the absence of any object or value.
- 'undefined' typically represents a variable that has been declared but not yet assigned a value.

What do you mean by REST API?

REST (Representational State Transfer) API (Application Programming Interface) is a set of web services that allow for the interaction between computer systems on the internet. It enables the interacting systems to communicate with each other by defining a set of rules and conventions. The interaction is stateless, meaning every HTTP request happens independently.

RESTful APIs are built using standard HTTP methods, which include GET, POST, PUT, DELETE, etc., and they are used to perform operations on resources, which are any kind of data, object, or service that can be accessed by the client.
